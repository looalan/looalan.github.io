  • DAYSWITH : String
    Offset to withdrawal date, such that (DTDUE – DAYSTOPAY) + (DAYSWITH) determines the date on which the funds are withdrawn from the user’s account.
  • DFLTDAYSTOPAY : String
    Default number of days to pay by check (except by transfer).
  • XFERDAYSWITH : String
    Number of days before processing date that funds are withdrawn for payment by transfer.
  • XFERDFLTDAYSTOPAY : String
    Default number of days to pay by transfer.
  • PROCDAYSOFF : List
    Days of week that no processing occurs; 0 or more of (MONDAY, TUESDAY, WEDNESDAY, THURSDAY, FRIDAY, SATURDAY, SUNDAY). <PROCDAYSOFF> indicate days to exclude when calculating dates that utilize other bill payment bits, such as <DAYSWITH> and <DFLTDAYSTOPAY> values.

  • PROCENDTM : String
    Time of day that day’s processing ends; used for determining whether today is still a processing day for calculation of due date. Clients should adjust this value to the current time zone, ignoring any date rollover that might occur. If the adjusted time is less than the current local time, processing for the day is over.
  • MODELWND : String
    Model window; the number of days before a recurring transaction is scheduled to be processed that it is instantiated on the system; <MODELWND>0 indicates that the server will always maintain a single spawned payment for each model.
