MessageSetCore

  • VER : String
    Version number of the message set, (for example, <VER>1 for version 1 of the message set).
  • URL : String
    URL where messages in this set are to be sent.
  • OFXSEC : String
    Security level required for this message set.
    Enum:
    NONE
    TYPE1
  • TRANSOSEC : String
    Y if transport-level security must be used, N if not used.
    Enum:
    Y
    N
  • SIGNONREALM : String
    Signon realm to use with this message set.
  • LANGUAGE : List
    One or more supported Language. If more than one language is supported, multiple <LANGUAGE> elements can be sent.
  • SYNCMODE : String
    FULL for full synchronization capability. LITE for lite synchronization capability.
    Enum:
    FULL
    LITE
  • REFRESHSUPT : String
    Y if server supports <REFRESH>Y within synchronizations. This option is irrelevant for full synchronization servers. Clients must ignore <REFRESHSUPT> (or its absence) if the profile also specifies <SYNCMODE>FULL. For lite synchronization, the default is N. Without <REFRESHSUPT>Y, lite synchronization servers are not required to support <REFRESH>Y requests.
    Enum:
    Y
    N
  • RESFILLER : String
    Y if server supports file-based error recovery.
    Enum:
    Y
    N
  • SPNAME : String
    Service provider name. Some financial institutions out-source their OFX servers to a service provider. In such cases, the SPNAME element should be included in the MSGSETCORE.
