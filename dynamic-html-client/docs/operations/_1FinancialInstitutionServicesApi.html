This is the API

getFIList

Queries all participating FI in the Trusted Network. Each FI in the list will be represented by a unique identifier within the Trusted Network.<br><br><input class='submit' type='submit' value='Fill in and try it out!' onclick='doFillInFIlist();'>

URL

https://qa-services.1fsapi.com/platform/trustednetwork/fs-api/filist

HTTP Method

GET

Response Type

FiDirectoryList

Parameters

  • Header: clientId String

    The FS-API client ID that will be used to validate this client.

  • Header: clientSecret String

    The FS-API client Secret that will be used together with the Client ID to validate this client.

  • Header: CorrelationID String

  • Query: query String

    User can choose FI List depending on what it supports (Account, Bank, Credit Card, Investment)

getProfile

This information includes general properties such as account types supported, user password requirements, specific messages supported, and how the client should batch requests and where to send the requests.<br><br><input class='submit' type='submit' value='Fill in and try it out!' onclick='doFillInFIprofile();'>

URL

https://qa-services.1fsapi.com/platform/trustednetwork/fs-api/fi/{fiDirId}/profile

HTTP Method

POST

Response Type

ProfileResponse

Parameters

  • Path: fiDirId String

    The FI Directory Id.

  • Header: CorrelationID String

    The FS-API client uses the correlationId request header to inform the server of an interaction tracing identifier. The server echoes the same correlationId in the response header to inform the FS-API client of the tracking id.
