getCreditCardAccountTransactions

The service queries transactions that are typically part of a regular paper statement. FS-API clients can retrieve the transactions for the specified Credit Card account.<br><br><input class='submit' type='submit' value='Fill in and try it out!' onclick='doFillInCreditCard();'>

URL

https://qa-services.1fsapi.com/platform/trustednetwork/fs-api/fi/{fiDirId}/account/credit

HTTP Method

POST

Response Type

CreditcardResponseMessageSetV1

Parameters

  • Query: query String

    If not specified the default value is TRANSACTIONS. The following are the expected values: TRANSACTIONS - for transaction history; BALANCEONLY - balance list only; and STATEMENTENDONLY - with statement end.

  • Body: body UserCreditCardRequest(UserCreditCardRequest)

    The request object that will hold the request body with the required request parameters in JSON or XML format.

  • Path: fiDirId String

    The FI Directory Id.

  • Query: start String

    A client can indicate the date range of transactions. dtStart is the date and time where the service began looking for the transaction history. The date can be an epoch, YYYY-MM-DD or YYYY-MM-DD HH:MM:SS format.

  • Query: end String

    A client can indicate the date range of transactions. dtEnd is the date and time where the service ended looking for the transaction history. The date can be an epoch, YYYY-MM-DD or YYYY-MM-DD HH:MM:SS format.

  • Header: CorrelationID String

    The FS-API client uses the correlationId request header to inform the server of an interaction tracing identifier. The server echoes the same correlationId in the response header to inform the FS-API client of the tracking id.

  • Header: clientId String

    The FS-API client ID that will be used to validate this client.

  • Header: clientSecret String

    The FS-API client Secret that will be used together with the Client ID to validate this client.

  • Header: Authorization String

    The OAuth Token.
